## How it works

RaTeX on Flutter uses Dart FFI to call the native Rust library, then renders the resulting DisplayList via a `CustomPainter` on Flutter's canvas.

```
LaTeX string
    ↓ RaTeXFfi.parseAndLayout()   [Dart FFI → libratex_ffi]
JSON DisplayList
    ↓ DisplayList.fromJson()       [Dart JSON decode]
DisplayList
    ↓ RaTeXPainter.paint()         [flutter/canvas]
CustomPaint Widget
```

## Installation

<Steps>
  <Step title="Add the dependency">
    Add `ratex_flutter` to your `pubspec.yaml`:

    ```yaml
    dependencies:
      ratex_flutter: ^0.0.15
    ```

    Then run:

    ```bash
    flutter pub get
    ```

    No native build is required — the published package includes prebuilt Android `.so` and iOS `RaTeX.xcframework`.

## Usage

<Tabs>
  <Tab title="Widget">
    `RaTeXWidget` is the recommended high-level API. It handles async parsing, sizing, and error display.

    ```dart
    import 'package:ratex_flutter/ratex_flutter.dart';

    class MathPage extends StatelessWidget {
      @override
      Widget build(BuildContext context) => Scaffold(
        body: Center(
          child: RaTeXWidget(
            latex: r'\frac{-b \pm \sqrt{b^2-4ac}}{2a}',
            fontSize: 28,
            onError: (e) => debugPrint('RaTeX: $e'),
          ),
        ),
      );
    }
    ```
  </Tab>
  <Tab title="Low-level CustomPainter">
    Use `RaTeXEngine` and `RaTeXPainter` when you need direct control over how and where the formula is painted:

    ```dart
    import 'package:ratex_flutter/ratex_flutter.dart';

    final dl      = RaTeXEngine.instance.parseAndLayout(r'\sum_{n=1}^\infty \frac{1}{n^2}');
    final painter = RaTeXPainter(displayList: dl, fontSize: 24);

    // In a CustomPaint widget:
    CustomPaint(painter: painter, size: Size(painter.widthPx, painter.totalHeightPx))
    ```
  </Tab>
  <Tab title="Inline formula">
    Use `RichText` with `WidgetSpan` to mix plain text and LaTeX in a single paragraph. The helper below parses `$...$` markers:

    ```dart
    import 'package:flutter/material.dart';
    import 'package:ratex_flutter/ratex_flutter.dart';

    /// Parses [text] with `$...$` inline math markers and returns a [RichText]
    /// that intermixes plain [TextSpan]s with [WidgetSpan]s.
    Widget buildInlineMath(String text, {double mathFontSize = 18, TextStyle? textStyle}) {
      final style = textStyle ??
          const TextStyle(fontSize: 16, height: 1.8, color: Colors.black87);

      final parts = text.split(r'$');
      final spans = <InlineSpan>[];

      for (int i = 0; i < parts.length; i++) {
        if (parts[i].isEmpty) continue;
        if (i.isEven) {
          // Plain text
          spans.add(TextSpan(text: parts[i], style: style));
        } else {
          // Inline math
          spans.add(WidgetSpan(
            alignment: PlaceholderAlignment.middle,
            baseline: TextBaseline.alphabetic,
            child: RaTeXWidget(
              latex: parts[i],
              fontSize: mathFontSize,
              onError: (e) => debugPrint('RaTeX inline error: $e'),
              loading: const SizedBox.shrink(),
            ),
          ));
        }
      }

      return RichText(text: TextSpan(children: spans));
    }

    // Usage:
    buildInlineMath(
      r'质能等价关系 $E = mc^2$，其中光速 $c \approx 3\times10^8\ \text{m/s}$。',
    )
    ```
  </Tab>
  <Tab title="Async (large formulas)">
    For large or complex formulas, offload parsing to a background isolate using `compute()`:

    ```dart
    import 'package:flutter/foundation.dart';

    final dl = await compute(
      (latex) => RaTeXEngine.instance.parseAndLayout(latex),
      r'\prod_{n=1}^\infty \left(1 - \frac{1}{n^2}\right)',
    );
    ```
  </Tab>
</Tabs>

## Coordinate system

All coordinates in the `DisplayList` are in **em units**, multiplied by `fontSize` (logical pixels) to get screen coordinates.

| Axis | Direction |
|------|-----------|
| X | Increases rightward from the left edge |
| Y | Increases downward from the top of the bounding box |
| Baseline | At Y = `height × fontSize` |

## Building from source

Use a local path dependency to develop against the repo directly.

<Steps>
  <Step title="Point pubspec.yaml at the local package">
    ```yaml
    dependencies:
      ratex_flutter:
        path: /path/to/RaTeX/platforms/flutter
    ```
  </Step>
  <Step title="Build native libraries">
    The local package does not include prebuilt binaries. Build them before running:

    | Platform | Command |
    |----------|---------|
    | iOS | `bash platforms/ios/build-ios.sh` |
    | Android | `bash platforms/android/build-android.sh` |

    **Prerequisites:** Flutter 3.10+, Dart 3.0+, Rust 1.75+.
  </Step>
</Steps>

## File map

| File | Purpose |
|------|---------|
| `pubspec.yaml` | Flutter plugin manifest |
| `ios/` | iOS plugin (podspec + `RaTeXPlugin.swift`); links `RaTeX.xcframework` |
| `android/` | Android plugin (`RaTeXPlugin.kt`); uses in-package `jniLibs` for `libratex_ffi.so` |
| `lib/ratex_flutter.dart` | Public API: `RaTeXEngine`, `RaTeXWidget` |
| `lib/src/display_list.dart` | Dart JSON types (`DisplayList`, `DisplayItem`, …) |
| `lib/src/ratex_ffi.dart` | Dart FFI bindings to `libratex_ffi` |
| `lib/src/ratex_painter.dart` | `CustomPainter` drawing loop |
