The C ABI is the primary integration point for native platforms. It exposes three functions from a single shared/static library built from `crates/ratex-ffi/include/ratex.h`.

<Note>
  The error state is thread-local. `ratex_parse_and_layout` and `ratex_get_last_error` are safe to call concurrently from multiple threads — each thread maintains its own last-error slot.
</Note>

## Header

```c ratex.h
#include "ratex.h"   // crates/ratex-ffi/include/ratex.h
```

---

## Functions

### `ratex_parse_and_layout`

Parse a LaTeX string and compute its display list as a JSON string.

```c
char* ratex_parse_and_layout(const char* latex);
```

<ParamField path="param.latex" type="const char*" required>
  Null-terminated UTF-8 LaTeX string. Must not be `NULL`.
</ParamField>

**Returns:** On success, a heap-allocated null-terminated JSON string describing the `DisplayList`. On failure, `NULL` — call `ratex_get_last_error()` for the reason. The caller is responsible for freeing the returned pointer with `ratex_free_display_list()`.

---

### `ratex_free_display_list`

Free the JSON string returned by `ratex_parse_and_layout`.

```c
void ratex_free_display_list(char* json);
```

<ParamField path="param.json" type="char*">
  Pointer returned by `ratex_parse_and_layout`. Passing `NULL` is a no-op.
</ParamField>

---

### `ratex_get_last_error`

Return the last error message produced on this thread.

```c
const char* ratex_get_last_error(void);
```

**Returns:** A null-terminated UTF-8 error string, or `NULL` if no error has occurred on this thread. The pointer is valid until the next call to `ratex_parse_and_layout` on the same thread.

<Warning>
  Do **not** free the pointer returned by `ratex_get_last_error`. It is owned by thread-local storage and is invalidated by the next call to `ratex_parse_and_layout` on this thread.
</Warning>

## Usage example

```c
#include <stdio.h>
#include "ratex.h"

const char* json = ratex_parse_and_layout("\\frac{1}{2}");
if (json) {
    // use json...
    ratex_free_display_list((char*)json);
} else {
    const char* err = ratex_get_last_error();
    fprintf(stderr, "RaTeX error: %s\n", err ? err : "(unknown)");
}
```

## DisplayList JSON format

The JSON string returned by `ratex_parse_and_layout` serializes the `DisplayList` structure. See the [DisplayList format guide](/guides/display-list) for the full schema. A brief outline:

```json
{
  "width":  1.234,
  "height": 0.700,
  "depth":  0.200,
  "items": [
    { "type": "GlyphPath", "x": 0.0, "y": 0.7, "scale": 1.0,
      "font": "Math-Italic", "char_code": 120, "color": {"r":0,"g":0,"b":0,"a":1} },
    { "type": "Line",  "x": 0.0, "y": 0.5, "width": 1.2, "thickness": 0.04,
      "color": {"r":0,"g":0,"b":0,"a":1} },
    { "type": "Rect",  "x": 0.0, "y": 0.0, "width": 1.0, "height": 0.5,
      "color": {"r":1,"g":0,"b":0,"a":1} },
    { "type": "Path",  "x": 0.0, "y": 0.0, "fill": true,
      "commands": [
        { "type": "MoveTo", "x": 0.0, "y": 0.0 },
        { "type": "LineTo", "x": 0.5, "y": 0.5 },
        { "type": "Close" }
      ],
      "color": {"r":0,"g":0,"b":0,"a":1} }
  ]
}
```

All coordinates are in **em units**. Multiply by your font size (pt or px) to get screen coordinates. The X axis increases rightward; the Y axis increases downward. The baseline sits at `y = height` measured from the top of the bounding box.

### PathCommand variants

| Variant | Fields |
|---------|--------|
| `MoveTo` | `x`, `y` |
| `LineTo` | `x`, `y` |
| `CubicTo` | `x1`, `y1`, `x2`, `y2`, `x`, `y` |
| `QuadTo` | `x1`, `y1`, `x`, `y` |
| `Close` | _(no fields)_ |

## Thread safety

`ratex_parse_and_layout` and `ratex_get_last_error` use **thread-local storage** for error state. They are safe to call concurrently from multiple threads without external synchronization. Each thread has its own independent last-error slot.
