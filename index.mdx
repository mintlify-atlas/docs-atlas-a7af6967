RaTeX is a LaTeX math rendering engine written entirely in Rust. It parses LaTeX, computes a layout, and produces a `DisplayList` — a platform-agnostic sequence of drawing commands that each target platform renders natively. No hidden WebView, no JavaScript runtime, no DOM overhead.

## Why RaTeX?

Every major cross-platform math renderer runs LaTeX through a browser or JavaScript engine — a hidden WebView consuming 50–150 MB RAM, startup latency before the first formula, and no offline guarantee.

RaTeX cuts the web stack out entirely:

| | KaTeX (web) | MathJax | **RaTeX** |
|---|---|---|---|
| Runtime | V8 + DOM | V8 + DOM | **Pure Rust** |
| Mobile | WebView | WebView | **Native** |
| Offline | Depends | Depends | **Yes** |
| Bundle overhead | ~280 kB JS | ~500 kB JS | **0 kB JS** |
| Memory | GC / heap | GC / heap | **Predictable** |
| Syntax coverage | 100% | ~100% | **~99%** |

## Get started

<Steps>
  <Step title="Choose your platform">
    Pick the integration guide for your target platform — iOS, Android, Flutter, React Native, or Web.
  </Step>
  <Step title="Install the package">
    Each platform has a published package. No native build required for most platforms.
  </Step>
  <Step title="Render a formula">
    Pass a LaTeX string to the platform widget or API. RaTeX handles parsing, layout, and drawing.
  </Step>
  <Step title="Explore advanced features">
    Use chemistry notation (`\ce`, `\pu`), SVG export, or the low-level DisplayList API for custom rendering.
  </Step>
</Steps>

<Note>
  RaTeX targets ~99% KaTeX syntax coverage. For the rare edge cases, see the [LaTeX syntax guide](/guides/latex-syntax) and the [live support table](https://erweixin.github.io/RaTeX/demo/support-table.html).
</Note>
